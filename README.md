# SW Maestro Busan Mentoring

AI·SW마에스트로 부산 멘토링 일정을 수집하고, 소마 페이지 안에서 일정표/달력/멘토별 보기로 확인하는 Chrome 확장 프로그램입니다.

소마 멘토링 페이지를 매번 열어 신청 상태를 확인하기 번거로워서 만든 도구입니다. 확장 아이콘을 누르면 부산 소마 페이지로 이동하면서 일정 화면이 바로 열리고, 소마 페이지에서는 오른쪽 아래 버튼으로도 열고 닫을 수 있습니다.

README의 확장 화면 캡처는 실제 개인정보나 신청 데이터를 쓰지 않고, 실제 확장 HTML/JS에 더미 데이터를 넣어 렌더링한 예시입니다. 일정 데이터는 화면 구성이 잘 보이도록 실제보다 많게 구성했습니다. 수집 진행 화면은 실제 `collector.js` 오버레이를 더미 응답으로 실행해 촬영했습니다.

## 할 수 있는 일

- 멘토링 일정과 상세 내용을 한 화면에서 보기
- 신청자 명단을 함께 수집해서 내 신청 완료 일정 표시
- 전체 일정 달력과 내 신청 일정 달력 보기
- 같은 시간에 신청한 일정이 있으면 충돌 경고 표시
- 관심/애매/제외 같은 개인 표시 저장
- 멘토 즐겨찾기와 멘토별 일정 보기
- 멘토 프로필을 Notion에서 자동 탐색해 다시 받아오는 멘토 갱신
- 데이터 갱신 시 신규 일정을 먼저 반영하고, 이어서 기존 일정 변경·삭제 비교
- 직전 대비 변경점(전체·신규·일정 변경·장소 변경·삭제)을 칩으로 분류해 안내
- 즐겨찾기한 멘토의 새 특강이 생기면 갱신 시 따로 알림
- 수집 결과를 파일로 다운로드하지 않고 확장 프로그램 IndexedDB에 저장
- 수집 데이터와 개인 표시를 백업 파일로 내보내고 복원

## 설치하기

이 저장소를 내려받습니다.

```bash
git clone https://github.com/Eomhyunjun/busan-soma-mentoring.git
```

Chrome에서 아래 순서로 설치합니다.

1. 주소창에 `chrome://extensions`를 입력합니다.
2. 오른쪽 위 `개발자 모드`를 켭니다.
3. `압축해제된 확장 프로그램을 로드`를 누릅니다.
4. 내려받은 저장소 안의 `extension` 폴더를 선택합니다.

```text
busan-soma-mentoring/extension
```

설치 후 Chrome 툴바의 확장 프로그램 목록에서 `SW Maestro Busan Mentoring`을 고정해두면, 아이콘만 눌러 바로 일정 화면을 열기 편합니다.

확장 아이콘을 누르면 팝업에서 부산 소마 페이지를 열거나, 로그인된 소마 탭에서 바로 멘토링 수집을 실행할 수 있습니다.

![확장 프로그램 팝업](docs/screenshots/popup.png)

## 데이터 수집하기

먼저 `https://www.swmaestro.ai/busan/sw/main/main.do`에 로그인되어 있어야 합니다. 로그인 쿠키가 있는 소마 탭에서만 멘토링 목록과 신청자 명단을 읽을 수 있습니다.

처음에는 수집된 데이터가 없으므로, 일정 화면을 연 뒤 오른쪽 위의 `데이터 갱신` 버튼을 눌러 수집합니다.

1. Chrome 툴바에서 확장 아이콘을 누릅니다. 소마 페이지가 아니면 부산 소마로 이동하면서 일정 화면이 열립니다.
2. 일정 화면 오른쪽 위의 `데이터 갱신`을 누릅니다.
3. 페이지 오른쪽 아래에 수집 진행 상태가 표시됩니다.
4. 수집이 끝나면 일정 화면이 새 데이터로 자동 갱신됩니다.

아래 캡처는 실제 `collector.js`가 만드는 진행 오버레이를 더미 응답으로 실행해 촬영한 예시입니다.

![데이터 수집 진행 상태](docs/screenshots/collector-progress-demo.png)

수집 결과는 파일로 내려받지 않고 확장 프로그램의 IndexedDB에 저장되며, 일정 화면은 이 데이터를 사용합니다.

로그인하지 않았거나 세션이 만료된 상태에서 수집을 시작하면, 확장이 소마 로그인/세션 오류 페이지를 감지해 아래 로그인 페이지로 이동합니다.

```text
https://www.swmaestro.ai/busan/sw/member/user/forLogin.do?menuNo=200025
```

## 일정 화면 열기

여는 방법은 두 가지입니다.

- **확장 아이콘**: Chrome 툴바의 확장 아이콘을 누르면, 소마 페이지가 아닐 때는 부산 소마로 이동하면서 일정 화면이 열리고, 이미 소마 페이지면 바로 열립니다.
- **떠 있는 버튼**: 소마 페이지 오른쪽 아래 quick menu 영역의 `멘토링 일정보기` 버튼으로 열고, 같은 버튼을 다시 누르면 원래 페이지로 돌아갑니다.

아래 일정 화면 캡처는 실제 `extension/schedule.html` 화면에 더미 일정과 멘토 데이터를 넣어 렌더링한 예시입니다.

![멘토링 일정 화면](docs/screenshots/schedule-calendar-demo.png)

## 일정 화면 사용법

상단의 탭으로 보는 방식을 바꿀 수 있습니다.

- `내 달력`: 기본 화면입니다. 신청 완료로 감지되었거나 내가 직접 신청 표시한 일정만 봅니다.
- `전체 달력`: 해당 월의 전체 멘토링 일정을 달력으로 봅니다.
- `일정`: 날짜별 목록입니다. 처음에는 모든 날짜가 접혀 있고, `전체 열기` / `전체 닫기`로 한 번에 펼치거나 접을 수 있습니다. 왼쪽 미니 달력에는 월 제목과 그달 건수(전체·신청), 이전/이번 달/다음 이동이 함께 들어 있습니다. 날짜 칸을 누르면 그 날짜가 펼쳐지며 그쪽으로 스크롤되고, 칸의 숫자는 그날 내가 신청한 건수입니다. `달력` 토글로 달력만 접을 수도 있습니다.
- `멘토`: 멘토 목록과 멘토 상세 일정을 봅니다. 멘토 프로필의 강의 목록에서 강의를 누르면 그 강의 상세로 바로 이동합니다.
- `내 멘토별`: 내가 신청한 일정을 멘토별로 묶어 봅니다.
- `같이 듣는 사람`: 신청자 명단 기준으로 나와 같은 강의를 듣는 사람을 봅니다.

`일정` 탭에서는 날짜별 목록과 왼쪽 미니 달력을 함께 봅니다. 날짜 칸을 누르면 해당 날짜 일정이 펼쳐지고, 일정 카드를 누르면 상세 모달로 들어갑니다.

![날짜별 일정 목록](docs/screenshots/schedule-list-demo.png)

상세 모달에서는 일정 기본 정보, 멘토, 신청자 명단, 개인 표시 버튼을 한 곳에서 확인합니다.

![멘토링 상세 모달](docs/screenshots/detail-modal-demo.png)

`멘토` 탭은 Notion에서 가져온 멘토 프로필과 연결된 멘토링 수를 보여주며, 별표로 즐겨찾기를 관리합니다.

![멘토 탭](docs/screenshots/mentor-tab-demo.png)

`내 멘토별` 탭은 내가 신청한 강의를 멘토 기준으로 묶어 보여줍니다.

![내 멘토별 탭](docs/screenshots/my-mentors-demo.png)

`같이 듣는 사람` 탭은 신청자 명단을 기준으로 나와 같은 강의를 듣는 사람과 겹치는 강의를 보여줍니다.

![같이 듣는 사람 탭](docs/screenshots/peers-demo.png)

필터는 현재 탭에서 실제로 동작하는 것만 표시됩니다. 예를 들어 `멘토` 탭에서는 검색만 보이고, `내 달력`에서는 월 이동만 보입니다.

탭과 필터 막대는 스크롤해도 화면 위쪽에 고정되며, `일정` 탭에서는 미니 달력도 그 바로 아래에 함께 고정됩니다.

일정 카드의 표시 버튼은 개인 메모용입니다.

- `신청`: 직접 신청한 것으로 표시합니다.
- `관심`: 나중에 다시 볼 일정으로 표시합니다.
- `애매`: 고민 중인 일정으로 표시합니다.
- `제외`: 보지 않을 일정으로 표시합니다.
- `표시한 것만`: 위 상태 중 하나라도 찍어둔 일정만 필터링합니다.

소마 신청자 명단에 내 이름이 `신청완료` 상태로 있으면, 마감된 일정이어도 `신청` 상태가 우선 표시됩니다.

## 충돌과 신청 상태 확인

일정 화면은 수집된 신청자 명단을 기준으로 내 신청 상태를 계산합니다. 소마 페이지에서 로그인한 사용자 이름을 감지하고, 상세 페이지 신청자 명단에 같은 이름이 `신청완료`로 있으면 해당 일정을 신청한 것으로 표시합니다.

신청한 일정끼리 시간이 겹치면 화면 상단에 시간 중복 경고가 표시됩니다. 경고 항목을 누르면 문제가 되는 일정 상세로 바로 이동할 수 있습니다.

![충돌과 신청 상태 표시](docs/screenshots/conflict-status-demo.png)

일정 카드에도 시간 관련 표시가 붙습니다.

- `충돌`: 내가 신청한 일정끼리 시간이 실제로 겹치는 경우
- `시간 겹침`: 이미 신청한 일정 때문에 같은 시간대에 듣기 어려운 다른 일정
- `다른 회차 신청함`: 같은 멘토/비슷한 제목의 다른 회차를 이미 신청한 경우

같은 날 여러 강의를 신청할 때는 `전체 달력`이나 `내 달력`에서 겹치는 시간을 한 번 더 확인하는 편이 좋습니다.

## 데이터 갱신하기

일정 화면 오른쪽 위의 `데이터 갱신` 버튼을 누르면 현재 소마 탭에서 다시 수집을 시작합니다.

갱신은 두 단계로 진행됩니다.

1. 현재 저장된 데이터의 가장 큰 멘토링 번호보다 큰 번호를 신규 일정으로 보고 먼저 수집합니다.
2. 신규 일정이 있으면 즉시 IndexedDB에 부분 저장하고, 화면에 먼저 보여줍니다.
3. 이어서 기존 번호의 멘토링을 다시 수집해 일정/장소 변경이나 삭제 여부를 최종 비교합니다.

이 방식은 새 멘토링이 생겼을 때 기다리는 시간을 줄이기 위한 것입니다. 진행 상태 모달은 크기가 고정되어 있고, 긴 모집명은 말줄임표로 표시됩니다.

최종 수집이 끝나면:

1. 최신 일정 데이터가 IndexedDB에 저장됩니다.
2. 열려 있던 일정 화면이 새 데이터를 다시 읽고 현재 뷰를 갱신합니다.
3. 새로 추가된 일정, 신청자 상태, 신청 인원 변경이 반영됩니다.

### 변경점 안내

화면을 열어둔 상태에서 갱신하면, 직전 수집본과 비교해 무엇이 바뀌었는지 화면 위쪽에 요약해 보여줍니다.

![데이터 갱신 변경점 안내](docs/screenshots/update-summary-demo.png)

- 변경 요약 배너: `전체` / `신규` / `삭제` / `일정 변경` / `장소 변경` 칩으로 묶어 보여줍니다. 처음에는 `전체`가 선택되어 모든 변경을 한 번에 봅니다.
- 일정·장소 변경은 "이전 → 이후"로 함께 표시하고, 항목을 누르면 해당 일정 상세로 이동합니다.
- 내가 신청한 일정의 변동(일정·장소 변경·삭제)은 맨 앞 "내 신청 변동"으로 따로 강조됩니다.
- 즐겨찾기한 멘토가 새 특강을 열면 별도의 강조 배너로 따로 알려줍니다.
- 바뀐 게 없으면 "변경 없음"만 잠깐 표시됩니다.

신청자 수나 마감 여부 같은 인원 관련 변화는 안내에 넣지 않고, 멘토링 정보 자체의 변화에만 집중합니다.

삭제는 "이전 데이터에는 있었지만 이번 최종 수집 결과에는 없는 멘토링 번호"로 판단합니다. 일정/장소 변경은 같은 멘토링 번호의 날짜·시간·장소 값이 달라졌는지 비교합니다.

처음 수집처럼 비교할 이전 데이터가 없으면 변경 안내가 표시되지 않습니다. 한 번 수집한 뒤 다시 갱신할 때부터 표시됩니다.

### 멘토 프로필 갱신

멘토 정보는 소마가 공개한 Notion 멘토 소개 페이지에서 받아와 확장 프로그램에 저장해두고 재사용합니다. 평소에는 저장해둔 정보를 그대로 쓰기 때문에, Notion에서 멘토 소개가 바뀌어도 화면에 바로 반영되지 않습니다.

`데이터 갱신` 옆 `멘토 갱신` 버튼을 누르면 저장해둔 정보를 무시하고 Notion에서 멘토 프로필을 다시 받아옵니다. 갱신이 끝나면 오른쪽 아래에 갱신된 멘토 수가 잠깐 표시되고, 즐겨찾기한 멘토는 그대로 유지됩니다.

![멘토 프로필 갱신](docs/screenshots/mentor-refresh-demo.png)

Notion 내부 컬렉션이나 뷰 ID가 바뀌어도, 확장은 루트 페이지에서 멘토 컬렉션을 자동으로 찾아 사용합니다. 멘토 프로필을 찾지 못하면 기존 캐시를 덮어쓰지 않아 빈 멘토 목록으로 바뀌는 일을 막습니다.

`멘토 갱신`은 멘토 프로필만 다시 받아오고 멘토링 일정 데이터는 건드리지 않습니다. 일정은 `데이터 갱신`, 멘토 소개는 `멘토 갱신`으로 나뉘어 있습니다.

## 백업과 복원

이 확장은 데이터를 확장 프로그램 단위 저장소(IndexedDB·chrome.storage.local)에 보관합니다. 평범한 새로고침이나 같은 폴더에서 다시 로드할 때는 데이터가 유지되지만, **다른 폴더 경로에서 로드하거나 다른 Chrome 프로필·다른 PC에서 열면** 빈 상태로 시작합니다. 저장소가 확장 ID에 묶여 있고, 압축해제 확장의 ID는 폴더 경로에서 만들어지기 때문입니다.

이럴 때를 대비해 일정 화면 오른쪽 위에 백업 버튼이 있습니다.

![백업과 복원](docs/screenshots/backup-restore-demo.png)

- `백업 내보내기`: 수집한 일정·멘토 정보와 관심/신청/즐겨찾기 표시를 JSON 파일 하나로 내려받습니다.
- `백업 가져오기`: 내보낸 JSON 파일을 선택하면 현재 데이터를 그 내용으로 덮어쓰고 화면을 다시 불러옵니다.

다른 PC로 옮기거나 재설치하기 전에 내보내두면, 새 환경에서 `백업 가져오기`로 그대로 복원할 수 있습니다.

## 업데이트하기

이미 설치한 확장 프로그램을 새 버전으로 바꿀 때는 제거 후 다시 설치할 필요가 없습니다.

1. 저장소 폴더에서 최신 코드를 받습니다.

```bash
git pull
```

2. `chrome://extensions`로 이동합니다.
3. 이 확장 프로그램 카드의 새로고침 버튼을 누릅니다.
4. 열려 있던 소마 페이지를 새로고침합니다.

설치할 때 선택하는 폴더는 저장소 루트가 아니라 `extension` 폴더입니다.

## 파일 구성

- `README.md`: 설치와 사용 설명
- `docs/screenshots/`: README에서 사용하는 스크린샷
- `extension/manifest.json`: Chrome 확장 프로그램 설정
- `extension/content.js`: 소마 페이지에 일정 보기 버튼과 iframe 주입, 아이콘 클릭 시 뷰어 열기
- `extension/content.css`: 소마 페이지에 붙는 floating 버튼/iframe 스타일
- `extension/background.js`: 아이콘 클릭 시 소마 이동·뷰어 열기, 수집 스크립트 실행과 IndexedDB 저장 처리
- `extension/collector.js`: 멘토링 목록, 상세, 신청자 명단 수집
- `extension/schedule.html`, `extension/schedule-app.js`: 일정 뷰어 화면

## 참고

- 이 확장은 `https://www.swmaestro.ai/busan/sw/main/main.do`에 로그인된 사용자 세션을 사용합니다.
- 신청자 명단은 상세 페이지 안의 페이지네이션을 따라가며 수집합니다.
- 내 이름은 소마 마이페이지의 로그인 사용자 표시에서 감지하고, 일정 화면 제목에 함께 표시합니다.
- 멘토링 번호(`qustnrSn`)는 신규 우선 수집과 기존 일정 비교 기준으로 사용합니다.
- 멘토 정보는 번들 파일을 포함하지 않고, 소마에서 공개한 Notion 멘토 소개 페이지를 런타임에 읽어옵니다.
- 소마 사이트 구조가 바뀌면 수집 로직도 수정이 필요할 수 있습니다.
